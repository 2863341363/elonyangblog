# Blog 重设计 PRD（简洁柔和版）

## 1. 项目背景

当前博客界面不符合预期，整体布局与视觉风格不够简洁，信息层级和阅读体验不够纯粹。
本次希望对博客前台进行一次整体重设计，弱化复杂装饰，突出内容本身，建立一个简洁、克制、柔和、耐看的个人博客界面。

## 2. 项目目标

### 2.1 核心目标
- 打造一个以内容阅读为核心的个人博客首页与文章页
- 页面结构简单清晰
- 配色柔和，不刺眼
- 视觉风格统一，减少多余装饰
- 首页可快速理解博客内容与分类
- 文章阅读体验舒适，适合长时间浏览
- 在桌面端和移动端都保持整洁

### 2.2 非目标
- 复杂动效
- 花哨插画和重装饰背景
- 多栏信息密集型门户布局
- 社交化功能重构
- 复杂运营模块

## 3. 目标用户
- 博客作者本人
- 访问博客阅读文章的普通访客
- 希望快速查看文章、分类、归档内容的读者

## 4. 产品定位
这是一个内容优先型个人博客，不是资讯门户，也不是视觉炫技型网站。

设计关键词：
- 简洁
- 柔和
- 留白
- 易读
- 安静
- 有个人感但不过度设计

## 5. 设计原则

### 5.1 内容优先
页面中的第一视觉焦点应当是文章标题、摘要、正文，而不是装饰元素。

### 5.2 减少干扰
减少无意义模块、重复信息、复杂边框、强动效、过多按钮。

### 5.3 柔和配色
整体色彩采用低饱和度方案，以浅色背景为主，强调温和、舒适、长期阅读不疲劳。

### 5.4 明确层级
通过字号、留白、颜色深浅、模块间距体现层级，不依赖复杂装饰。

### 5.5 响应式优先
移动端保持简洁、单列、易点击。

## 6. 核心页面

### 6.1 首页
- 顶部导航：首页、文章、分类、关于
- 顶部简介区域：博客名、一句简介、一小段个人说明
- 最新文章列表：标题、日期、摘要、分类标签
- 以单栏主内容结构为主，桌面端允许轻侧栏

### 6.2 文章页
- 标题突出
- 元信息简洁展示
- 正文区域宽度适中
- 行高舒适
- 代码块、引用、图片风格统一
- 上下篇导航和返回入口清晰但不抢眼

### 6.3 分类页
- 分类名称清晰展示
- 按分类列出文章
- 不做复杂视觉卡片墙

### 6.4 关于页
- 自我介绍
- 博客记录方向
- 联系方式 / GitHub / 社交链接
- 纯内容排版优先

## 7. 视觉设计要求

### 7.1 整体风格
- 极简
- 柔和
- 温暖
- 安静
- 轻文艺但不过度装饰

避免：
- 高饱和撞色
- 强烈渐变
- 复杂玻璃拟态
- 厚重阴影
- 大面积纯黑纯白强对比

### 7.2 配色方向
- 背景色：暖白、米白、浅奶油色
- 主文字：深灰而非纯黑
- 次级文字：中灰
- 边框：极浅灰
- 强调色：雾蓝 / 鼠尾草绿 / 淡紫灰

### 7.3 字体与排版
- 字体清晰、现代、耐读
- 标题与正文层级分明
- 正文不宜过小
- 行高适中偏大
- 页面留白明显

## 8. 交互要求
保留：
- 导航 hover / active 状态
- 卡片轻微 hover
- 链接 hover
- 分页切换
- 移动端菜单展开收起

不建议加入：
- 大幅动画
- 滚动视差
- 自动轮播
- 打断阅读的弹窗

## 9. 验收标准

### 视觉层面
- 整体明显比旧版更简洁
- 配色柔和统一
- 页面不显拥挤
- 文章阅读舒适

### 结构层面
- 首页、文章页、分类页、关于页结构统一
- 导航清晰，入口明确
- 桌面端与移动端都可正常使用

### 体验层面
- 首页可快速浏览最新文章
- 文章页阅读不受干扰
- 不存在明显视觉噪音和冗余装饰
