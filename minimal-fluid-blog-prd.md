# ElonYang 的博客极简风格 PRD

## 1. 产品概述

### 1.1 产品名称

`ElonYang 的博客`

### 1.2 产品定位

一个单人个人博客，用于沉淀 ElonYang 的学习记录、技术笔记、生活观察和阶段性思考。站点采用简洁、克制、偏 Hexo / Fluid 风格的界面语言：深色顶部导航与大面积 Hero 区建立沉静气质，浅米色背景和居中的白色内容卡片承载阅读内容。

### 1.3 设计目标

- 让博客第一眼呈现“干净、沉稳、专注阅读”的感觉。
- 保留个人博客的轻量感，不做复杂运营型门户。
- 用统一的深色 Hero、白色卡片和柔和阴影建立稳定视觉系统。
- 优先保证文章阅读、文章发现和内容归档体验。
- 首版不考虑多人作者、投稿、用户登录、后台管理等功能。

### 1.4 目标用户

- ElonYang：发布和管理自己的学习、技术、生活类文章。
- 普通读者：快速阅读文章、按分类或标签查找内容。
- 合作或关注者：通过关于页了解 ElonYang 的背景、方向和联系方式。

## 2. 核心设计方向

### 2.1 视觉关键词

- 简洁：页面结构清楚，元素数量少。
- 沉稳：深灰顶部区域，突出标题和站点气质。
- 留白：正文卡片内部保持宽松间距。
- 轻量：不使用复杂插画和强装饰。
- 专注：内容卡片是页面视觉中心。

### 2.2 参考风格总结

页面由三层构成：

1. 深灰色顶部导航栏。
2. 深灰色 Hero 区，中心展示博客标语。
3. 浅米色页面背景上叠放一张白色内容卡片。

整体布局应避免复杂侧栏、彩色卡片、人物头像、卡通插画。视觉重点放在“品牌 + 标语 + 内容卡片”。

## 3. 信息架构

### 3.1 顶部导航

导航固定在页面顶部，不一定吸顶，但所有页面保持一致。

导航内容：

- 左侧：`ElonYang 的博客`
- 右侧：首页
- 右侧：文章
- 右侧：分类
- 右侧：标签
- 右侧：关于
- 右侧：主题切换图标

### 3.2 页面结构

首版包含以下页面：

- 首页
- 文章列表页
- 文章详情页
- 分类页
- 标签页
- 关于页
- 搜索页
- 404 页面

## 4. 全局布局规范

### 4.1 页面外层

- 页面背景使用浅米色：`#fbf4e8` 或接近色。
- 顶部导航和 Hero 使用深灰：`#3c3b38`。
- 主内容卡片使用白色：`#ffffff`。
- 内容最大宽度建议为 `1100px` 到 `1180px`。
- 卡片圆角建议 `8px`。
- 卡片阴影要明显但柔和，类似：

```css
box-shadow: 0 24px 48px rgba(40, 35, 28, 0.22);
```

### 4.2 顶部导航

导航高度建议 `64px`。

视觉要求：

- 深灰背景。
- 左侧博客名加粗。
- 菜单项为白色或接近白色。
- 菜单项可带小图标，但图标必须小而克制。
- 当前页面菜单可通过颜色加亮、下划线或轻微加粗表示。

### 4.3 Hero 区

Hero 区是所有主页面的统一视觉入口。

通用结构：

- 深灰背景。
- 高度建议 `380px` 到 `420px`。
- 中心展示页面标题或站点标语。
- 字体为白色，字号建议 `32px` 到 `42px`。
- 首页文案：`记录学习、技术和生活_`
- 其他页面可替换为页面标题，例如 `文章归档_`、`分类_`、`关于我_`。

### 4.4 内容卡片

内容卡片叠放在 Hero 下方，形成参考图中的“浮层”效果。

布局要求：

- 卡片顶部与 Hero 底部重叠，向上偏移约 `40px` 到 `70px`。
- 卡片宽度建议 `70%` 到 `78%` 视口宽度，最大 `1120px`。
- 卡片最小高度根据页面内容自适应。
- 卡片内部左右 padding 建议 `96px` 到 `120px`，移动端降为 `24px`。

## 5. 页面详细需求

### 5.1 首页

#### 页面目标

让访问者快速理解这是 ElonYang 的个人博客，并进入最新文章。

#### 页面结构

- 顶部导航
- Hero 标语
- 中央白色内容卡片
- 最新文章预览
- 页脚

#### 内容卡片模块

- 欢迎语：例如 `你好世界`
- 日期：显示最近更新日期
- 最新文章标题
- 最新文章摘要
- 发布时间
- 可展示 2 到 3 篇文章，但保持列表简洁。

#### 设计要求

- 首页不需要复杂插画。
- 不需要右侧栏。
- 不需要大面积彩色标签。
- 文章预览之间使用留白或细分割线区分。

### 5.2 文章列表页

#### 页面目标

展示所有文章，让读者快速浏览标题、摘要和发布时间。

#### 页面结构

- 顶部导航
- Hero 标题：`文章归档_` 或 `全部文章_`
- 中央内容卡片
- 文章列表
- 分页

#### 文章列表项字段

- 文章标题
- 文章摘要
- 发布时间
- 分类
- 标签

#### 设计要求

- 文章列表采用纵向排列。
- 每篇文章之间用细线分隔。
- 标题字号建议 `22px` 到 `28px`。
- 摘要颜色使用蓝灰或中灰。
- 日期和分类信息使用小字号。
- 不使用右侧搜索卡、人物头像卡或订阅卡。

### 5.3 文章详情页

#### 页面目标

提供安静、沉浸、长时间阅读友好的文章页。

#### 页面结构

- 顶部导航
- Hero 显示文章标题或统一文案
- 中央内容卡片
- 文章标题
- 发布时间、分类、标签
- 正文
- 上一篇 / 下一篇
- 返回文章列表入口

#### 正文样式

- 正文字号建议 `16px` 到 `18px`。
- 行高建议 `1.8`。
- 正文最大宽度建议 `760px` 到 `860px`。
- 代码块使用浅灰背景。
- 引用块使用左边框或浅色背景。
- 图片宽度不超过正文区域。

#### 设计要求

- 不需要复杂目录侧栏。
- 不需要浮动点赞、收藏、分享按钮。
- 文章页重点是阅读内容。

### 5.4 分类页

#### 页面目标

帮助读者按主题浏览文章。

#### 页面结构

- 顶部导航
- Hero 标题：`分类_`
- 中央内容卡片
- 分类列表

#### 分类项字段

- 分类名称
- 分类描述
- 文章数量

#### 示例分类

- 技术笔记
- 学习记录
- 工具效率
- 生活随笔
- 项目复盘

#### 设计要求

- 分类项可用简洁列表或两列网格。
- 不使用花哨图标。
- 分类数量少时保持居中排版。

### 5.5 标签页

#### 页面目标

通过关键词帮助读者快速发现相关内容。

#### 页面结构

- 顶部导航
- Hero 标题：`标签_`
- 中央内容卡片
- 标签云或标签列表

#### 标签样式

- 标签使用浅灰底或无背景文字链接。
- 标签字号可根据文章数量轻微变化。
- 标签之间保持足够间距。

#### 设计要求

- 不使用过多颜色。
- 标签页整体要轻，不能像运营后台。

### 5.6 关于页

#### 页面目标

介绍 ElonYang 的个人背景、写作方向和联系方式。

#### 页面结构

- 顶部导航
- Hero 标题：`关于我_`
- 中央内容卡片
- 自我介绍
- 写作方向
- 联系方式

#### 内容建议

自我介绍：

`你好，我是 ElonYang。这里记录我的学习路线、技术笔记和日常思考。`

写作方向：

- 前端开发
- 编程学习
- 效率工具
- 项目部署
- 生活记录

联系方式：

- GitHub
- Email
- X / Twitter

#### 设计要求

- 不做团队介绍。
- 不放多作者头像。
- 可放一张简单头像，但不是必须。
- 语气要自然，不像企业官网。

### 5.7 搜索页

#### 页面目标

帮助读者按关键词查找文章。

#### 页面结构

- 顶部导航
- Hero 标题：`搜索_`
- 中央内容卡片
- 搜索输入框
- 搜索结果列表
- 空状态

#### 搜索输入框

- 输入框宽度占内容卡片大部分。
- 占位文案：`输入关键词搜索...`
- 支持回车搜索。

#### 搜索结果字段

- 文章标题
- 命中的摘要片段
- 发布时间
- 分类 / 标签

#### 空状态文案

`没有找到相关内容，换个关键词试试。`

### 5.8 404 页面

#### 页面目标

在访问不存在页面时保持友好，并引导返回。

#### 页面结构

- 顶部导航
- Hero 标题：`404_`
- 中央内容卡片
- 错误说明
- 返回首页按钮
- 查看文章按钮

#### 文案建议

标题：

`页面没有找到`

说明：

`也许链接已经移动，先回到首页继续阅读。`

按钮：

- 返回首页
- 查看文章

## 6. 组件规范

### 6.1 内容卡片

用途：

- 首页最新内容
- 文章列表
- 文章正文
- 分类与标签
- 关于页内容

样式：

- 背景：白色
- 圆角：8px
- 阴影：柔和但可见
- 内边距：桌面端宽松，移动端收紧

### 6.2 文章列表项

样式：

- 标题加粗。
- 摘要颜色比标题浅。
- 元信息使用小字号。
- 列表项之间用细线分隔。

### 6.3 按钮

按钮数量应少。

类型：

- 主按钮：返回首页、查看文章、提交搜索。
- 文本按钮：继续阅读、查看分类。

样式：

- 可使用深灰描边或深色填充。
- 圆角建议 `6px` 到 `8px`。
- 不使用高饱和颜色。

### 6.4 图标

图标使用原则：

- 只用于导航和元信息。
- 图标尺寸小。
- 风格统一。
- 不使用复杂插画图标。

## 7. 交互需求

### 7.1 导航

- 点击菜单进入对应页面。
- 当前菜单项需要有选中状态。
- 移动端导航可折叠为菜单按钮。

### 7.2 文章跳转

- 点击文章标题进入详情页。
- 文章详情页可返回文章列表。

### 7.3 搜索

- 输入关键词后展示结果。
- 搜索为空时不展示错误。
- 无结果时展示空状态文案。

### 7.4 主题切换

可保留主题切换图标。

首版要求：

- 图标存在即可。
- 不强制实现完整暗色模式。
- 若实现暗色模式，需要保证文章正文可读性。

## 8. 响应式要求

### 8.1 桌面端

- 顶部导航水平排列。
- Hero 高度稳定。
- 内容卡片居中，宽度不超过最大内容宽度。

### 8.2 平板端

- 内容卡片宽度扩大到屏幕宽度的 `86%` 到 `92%`。
- 卡片内边距减少。
- 导航间距减少。

### 8.3 移动端

- 导航折叠或换行。
- Hero 高度降低。
- 内容卡片不再大幅上移，避免遮挡。
- 文章列表单列显示。
- 正文字号不低于 `16px`。

## 9. SEO 与性能

### 9.1 SEO

- 首页、文章页、分类页、标签页、关于页都需要独立 title。
- 文章详情页需要 description。
- 使用语义化标题结构。
- 保留 Hexo 的静态生成优势。

### 9.2 性能

- 首版尽量不引入大型前端框架。
- 不使用大量图片或动画。
- CSS 保持轻量。
- 页面应可静态部署到 Cloudflare Pages、GitHub Pages 或类似平台。

## 12. 验收标准

- 页面第一屏与参考图一致：深色导航、深色 Hero、米色背景、中央白色卡片。
- 首页没有复杂侧栏、人物卡片或大面积彩色模块。
- 所有页面保持同一套视觉结构。
- 文章列表清晰展示标题、摘要和日期。
- 文章详情页适合长时间阅读。
- 移动端无横向滚动、文字遮挡或卡片溢出。
- 构建命令成功执行。
- 本地预览和部署后的静态页面一致。

## 13. 开发实现建议

如果基于 Hexo 实现：

1. 保留 Hexo 作为静态站点框架。
2. 使用主题配置或自定义 generator 输出页面。
3. 将核心样式集中在一个自定义 CSS 文件中。
4. 避免同时维护多套视觉风格。
5. 本地预览建议使用静态模式，确保看到的是 `public` 产物。

建议命令：

```bash
npm run build
npm run server -- --port 4000
```

若使用自定义静态预览，需要确保预览的是构建后的 `public` 目录。
